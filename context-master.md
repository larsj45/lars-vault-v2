# Context Master

This file is the top-level context entry point for AI agents and collaborators.

## Purpose

Use this document to understand the operating context before working on projects, prompts, workflows, or strategic decisions.

## Operating Model

The goal is to build a Markdown-first, AI-readable operating system that supports:

- business building
- AI-assisted execution
- international expansion
- reusable workflows
- autonomous agents
- technical systems
- decision making

## Core Belief

AI tools become more powerful when they operate with durable, structured, portable context.

This vault exists to provide that context.

## Main Knowledge Areas

- operational philosophy
- active projects
- tech and AI stack
- prompt library
- agent instructions
- venture playbooks
- market validation workflows
- decision logs

## How AI Agents Should Use This Vault

1. Start here.
2. Read `operational-philosophy.md`.
3. Read `active-projects.md`.
4. Check the relevant project folder.
5. Use playbooks before inventing new workflows.
6. Add useful outputs back into the vault.
7. Keep all writing clear, simple, and reusable.
